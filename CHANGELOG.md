## [v0.1.0] - 2026-06-28

### Highlights

- Generic Go framework for creating, validating, and managing standards catalogs
- CLI tool for validating, querying, and displaying catalog data
- Comprehensive validation system with configurable options

### Added

- Core catalog types: `Catalog`, `Standard`, `CatalogMetadata`, `Implementation`, `Maintainer`
- Enum types for type safety: `StandardStatus` (draft, proposed, adopted, deprecated, retired)
- Enum types: `Category` (authentication, authorization, identity, provisioning, communication, discovery, governance)
- Enum types: `IdentityLayer` (human, agent, workload, service)
- Enum types: `ImplementationStatus` (experimental, beta, stable, deprecated)
- `Load()` function to load catalog from JSON/YAML file
- `Parse()` function to parse raw bytes with format detection
- `Save()` function to save catalog to file
- `LoadAndValidate()` function for combined load and validate operation
- `FindByID()` method to lookup standard by identifier
- `FindByCategory()` method to filter by category
- `FindByLayer()` method to filter by identity layer
- `FindByStatus()` method to filter by status
- `FindByTag()` method to filter by tags
- `FindByOrganization()` method to filter by organization
- `GetOrganizations()` method to get all unique organizations
- `GetTags()` method to get all unique tags
- Validation: required field validation
- Validation: ID format validation (lowercase, hyphens allowed)
- Validation: URL validity checking
- Validation: enum value validation for status, category, layer
- Validation: cross-reference validation (compatibleWith, supersedes, supersededBy)
- Validation: ID uniqueness validation within catalog
- Validator option: `RequireImplementations` to require at least one implementation
- Validator option: `RequireDescription` to require descriptions
- Validator option: `AllowedOrganizations` to restrict allowed organizations
- Validator option: `AllowedCategories` to restrict allowed categories
- Validator option: `AllowedLayers` to restrict allowed layers
- CLI `validate` command to validate a catalog file
- CLI `list` command to display all standards in tabular format
- CLI `show` command to display standard details as JSON
- CLI `query` command with flags: --category, --layer, --status, --org, --tag
- CLI `stats` command to show catalog statistics
- Example catalog files in testdata/

### Tests

- Comprehensive unit tests with 96%+ code coverage
