# v0.1.0 Release Notes

**Release Date:** 2026-06-28

## Highlights

- Generic Go framework for creating, validating, and managing standards catalogs
- CLI tool for validating, querying, and displaying catalog data
- Comprehensive validation system with configurable options

## Features

### Core Types

The framework provides a complete type system for standards catalogs:

- `Catalog` - Collection of standards with metadata
- `Standard` - Individual standard entry with comprehensive fields
- `CatalogMetadata` - Name, version, description, maintainers
- `Implementation` - Known implementations with language/status
- `Maintainer` - Contact information

### Enum Types

Type-safe enums for consistent data:

| Type | Values |
|------|--------|
| `StandardStatus` | draft, proposed, adopted, deprecated, retired |
| `Category` | authentication, authorization, identity, provisioning, communication, discovery, governance |
| `IdentityLayer` | human, agent, workload, service |
| `ImplementationStatus` | experimental, beta, stable, deprecated |

### File I/O

- `Load()` - Load catalog from JSON/YAML file
- `Parse()` - Parse raw bytes with format detection
- `Save()` - Save catalog to file
- `LoadAndValidate()` - Combined load and validate operation

### Query Methods

Filter and search standards programmatically:

- `FindByID()` - Lookup standard by identifier
- `FindByCategory()` - Filter by category
- `FindByLayer()` - Filter by identity layer
- `FindByStatus()` - Filter by status
- `FindByTag()` - Filter by tags
- `FindByOrganization()` - Filter by organization
- `GetOrganizations()` - Get all unique organizations
- `GetTags()` - Get all unique tags

### Validation System

Comprehensive validation with detailed error reporting:

- Required field validation
- ID format validation (lowercase, hyphens allowed)
- URL validity checking
- Enum value validation for status, category, layer
- Cross-reference validation (compatibleWith, supersedes, supersededBy)
- ID uniqueness validation within catalog

### Configurable Validator Options

- `RequireImplementations` - Require at least one implementation per standard
- `RequireDescription` - Require descriptions for all standards
- `AllowedOrganizations` - Restrict to specific organizations
- `AllowedCategories` - Restrict to specific categories
- `AllowedLayers` - Restrict to specific identity layers

### CLI Tool

The `standards-catalog` CLI provides five commands:

```bash
# Validate a catalog file
standards-catalog validate catalog.yaml

# List all standards in tabular format
standards-catalog list catalog.yaml

# Show details of a specific standard
standards-catalog show catalog.yaml oauth2

# Query with filters
standards-catalog query catalog.yaml --layer=agent --category=authorization

# Show catalog statistics
standards-catalog stats catalog.yaml
```


## Library Usage

```go
import "github.com/grokify/standards-catalog-framework/catalog"

// Load and validate
c, errors, err := catalog.LoadAndValidate("catalog.yaml")
if err != nil {
    log.Fatal(err)
}
if errors.HasErrors() {
    log.Fatal(errors)
}

// Query standards
agentStandards := c.FindByLayer(catalog.LayerAgent)
authStandards := c.FindByCategory(catalog.CategoryAuthentication)
```
